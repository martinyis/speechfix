    1. Create `hooks/useRecording.ts` -- custom hook wrapping expo-audio:
       - Import { useAudioRecorder, AudioModule, RecordingPresets } from 'expo-audio'
       - Import { useState, useCallback, useRef, useEffect } from 'react'
       - The hook returns: { isRecording, startRecording, stopRecording, audioUri, duration, meteringValues }
       - On mount, request audio permissions: AudioModule.requestRecordingPermissionsAsync()
       - startRecording():
         - Call audioRecorder.prepareToRecordAsync(RecordingPresets.HIGH_QUALITY) -- produces .m4a/AAC
         - Call audioRecorder.record()
         - Set isRecording = true
         - Start a metering interval (setInterval every 100ms) that reads audioRecorder.currentMetering and appends to meteringValues array (for waveform). Limit array to last 50 values.
       - stopRecording():
         - Call audioRecorder.stop()
         - Set isRecording = false
         - Clear metering interval
         - Set audioUri = audioRecorder.uri
         - Set duration from audioRecorder.currentTime (seconds)
       - Clean up interval on unmount
       - Note: expo-audio's useAudioRecorder provides the recorder instance. Check the actual expo-audio API -- if useAudioRecorder returns the recorder directly, use its methods. If the API differs from this description, adapt accordingly but keep the same hook interface (isRecording, startRecording, stopRecording, audioUri, duration, meteringValues).

    2. Create `components/RecordButton.tsx`:
       - Large circular button, 120x120 pixels (per CONTEXT.md: "large centered mic button -- the dominant element")
       - When not recording: dark circle (#1a1a1a) with white mic icon (use a simple SVG path or Unicode mic character as placeholder, or use @expo/vector-icons Ionicons 'mic' icon if available in the template)
       - When recording: red circle (#ff3b30) with white square stop icon (Ionicons 'stop' or a 40x40 white rounded square)
       - Props: { isRecording: boolean; onPress: () => void }
       - Use Pressable with opacity feedback on press
       - Add a subtle scale animation on press using Animated API (scale to 0.95 on press in, back to 1.0 on release)
       - Style: width 120, height 120, borderRadius 60, alignItems 'center', justifyContent 'center'

    3. Create `components/Waveform.tsx`:
       - Visualizes audio metering values as vertical bars (per CONTEXT.md: "live waveform visualization while recording")
       - Props: { meteringValues: number[]; isActive: boolean }
       - Display a row of vertical bars, each bar's height proportional to the metering value
       - Metering values from expo-audio are in dBFS (negative, typically -160 to 0). Normalize: height = Math.max(((value + 160) / 160) * maxBarHeight, 4) where maxBarHeight is ~80
       - Bar count: render the meteringValues array (last 50 values), each bar is 4px wide with 2px gap
       - Bar color: white (#fff) with opacity 0.8
       - When isActive is false, render nothing (waveform disappears when recording stops per CONTEXT.md)
       - Container: height 100, width auto (expands to fit bars), centered horizontally
       - Animate new bars appearing from the right side using the Animated API or simply re-render (the visual effect of new bars appearing is sufficient)
