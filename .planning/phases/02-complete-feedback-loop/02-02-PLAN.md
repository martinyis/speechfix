**Progressive display strategy (per CONTEXT.md INFRA-05 decision):**

The POST /sessions response from Plan 01 returns everything at once (transcription + corrections + fillerWords). But the user wants to see transcription FIRST, then corrections appearing after. The approach:

1. After recording, `index.tsx` navigates to `/results?sessionId=X` immediately after upload completes
2. The upload response already contains everything. Pass both `sentences` and the full analysis data to the results screen.
3. Results screen shows sentences as plain text first, displays "Analyzing speech..." banner, then after a brief delay (simulating progressive feel) fades in the corrections/filler chips.

Actually, for true progressive display: modify the POST /sessions route flow on the mobile side. The backend already returns everything together. To get progressive UX:
- Option A: Two-phase response (complex backend change)
- Option B: Navigate to results with just `sessionId` and `sentences` immediately. Then poll GET /sessions/:id for the analysis. But the backend does analysis synchronously in POST...
- Option C (simplest, best UX): Navigate to results immediately with sentences from the POST response. Show the banner. After a brief artificial delay (500ms) or use Animated API to fade in corrections that came with the same response.

**Go with Option C** -- the POST already takes time for Claude analysis. The user sees the loading overlay on the record screen during that time. When it completes, they navigate to results and see everything. But to give the progressive FEEL: show sentences first with the banner for 800ms, then animate in the corrections.

**Create mobile/hooks/useSession.ts:**

A hook to fetch session detail by ID (used by history detail screen later, and as fallback):
```typescript
import { useQuery } from '@tanstack/react-query';
import { API_BASE_URL } from '../lib/api';
import { SessionDetail } from '../types/session';

export function useSession(sessionId: number | null) {
  return useQuery({
    queryKey: ['session', sessionId],
    queryFn: async (): Promise<SessionDetail> => {
      const res = await fetch(`${API_BASE_URL}/sessions/${sessionId}`);
      if (!res.ok) throw new Error('Failed to fetch session');
      const data = await res.json();
      return data.session;
    },
    enabled: sessionId !== null,
  });
}
```
